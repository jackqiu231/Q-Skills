## 快速开始
1. 先配置好X平台开发者账号获取的API密钥和访问令牌，设置到环境变量中
2. 调用技能传入要发布的推文内容即可发布

## 参数说明
| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| text | string | 是 | 要发布的推文内容，长度需符合X平台限制（最多280字符） | 无 |
| media_paths | array[string] | 否 | 要附带的本地图片/视频文件路径列表，最多支持4个媒体文件 | 无 |

## 出参说明
返回JSON包含以下字段：
- success: bool，是否发布成功
- output: string，成功时返回推文URL，失败时返回错误信息
- data: dict，成功时包含完整的推文响应数据

## 使用示例
### 纯文本推文
```json
{
  "text": "这是一条测试推文，来自AI助理自动发布"
}
```

### 带图片的推文
```json
{
  "text": "今天的西湖风景好美~",
  "media_paths": ["/Users/bytedance/mycowork/images/1772430469333.png"]
}
```

## 环境变量（二选一配置即可）
### OAuth1.0a认证（推荐，支持纯文本和带媒体的推文）
- `X_API_KEY`: X平台开发者API Key
- `X_API_SECRET`: X平台开发者API Secret
- `X_ACCESS_TOKEN`: X平台用户访问令牌
- `X_ACCESS_TOKEN_SECRET`: X平台用户访问令牌密钥

### OAuth2.0 Bearer Token认证（仅支持纯文本推文）
- `X_BEARER_TOKEN`: X平台用户级Bearer Token，需要在开发者后台申请`tweet.write`权限

## 错误处理
| 错误现象 | 原因 | 解决方案 |
|----------|------|----------|
| 认证失败 | API密钥或访问令牌错误 | 检查环境变量配置的密钥是否正确，确保账号有读写权限 |
| 内容过长 | 推文内容超过280字符限制 | 缩短推文内容再重试 |
| 媒体文件不存在 | 传入的媒体路径无效 | 检查文件路径是否正确，确保文件存在 |

## 注意事项
- 需要先在X平台开发者后台创建应用并申请OAuth 1.0a权限，获取对应的密钥
- 发布内容需遵守X平台的社区准则和使用条款
